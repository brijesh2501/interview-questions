## Interview Tips for Projects

### Before Starting

1. **Clarify Requirements** - Ask clarifying questions
2. **Define Scope** - What's in/out of scope?
3. **Discuss Architecture** - Talk through design
4. **Ask About Constraints** - Time, performance, scale
5. **Confirm Success Criteria** - How will it be evaluated?

### During Development

1. **Talk Through Your Thinking** - Communicate your approach
2. **Use Design Patterns** - Apply known patterns
3. **Write Clean Code** - Readability matters
4. **Handle Errors** - Graceful error handling
5. **Think About Scale** - How does it scale?

### After Development

1. **Review Your Code** - Quality check
2. **Test Thoroughly** - Edge cases
3. **Optimize if Needed** - Performance improvements
4. **Document Key Decisions** - Explain trade-offs
5. **Be Ready to Discuss** - Scalability, improvements
